# Standard Operating Procedure (SOP)
## Schedule Upload and Verification — Transport Management System

---

## Purpose
To ensure schedule data is uploaded accurately into the Transport Management System and verified before being used by dispatch and operations teams.

---

## Scope
This SOP applies to:
- Scheduling staff
- Dispatch supervisors
- Application / Operations support personnel

---

## Responsibilities
- **Scheduler:** Prepares and submits schedule data
- **Application Support:** Uploads, validates, and verifies data
- **Dispatch Supervisor:** Confirms operational readiness

---

## Prerequisites
- Approved schedule file (Excel format)
- Access to Transport Management System (admin role)
- User access for dispatch verification

---

## Procedure

### Step 1 — Pre-Upload Validation
- Review schedule file for:
  - Correct date ranges
  - Complete route and driver assignments
  - Proper file format and required fields
- Correct any formatting or data issues before upload

---

### Step 2 — Upload Schedule Data
- Log in to the Transport Management System with admin credentials
- Navigate to the schedule import section
- Upload the validated schedule file
- Monitor upload status for errors or warnings

---

### Step 3 — Post-Upload System Check
- Review system messages or logs related to the import
- Confirm no validation or synchronization errors occurred

---

### Step 4 — User Verification
- Log in using a dispatcher account
- Confirm:
  - Routes are visible
  - Driver assignments are accurate
  - No missing or duplicated entries exist

---

### Step 5 — Operational Confirmation
- Notify the dispatch supervisor that schedules are live
- Confirm readiness for daily operations

---

## Error Handling
If issues are detected:
- Stop operational use of the affected schedules
- Refer to the Troubleshooting Runbook
- Correct data and re-upload as required

---

## Documentation
- Record any issues and resolutions in an Incident Report
- Update this SOP if recurring issues are identified
